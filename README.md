# Фуршетное меню

Сайт с ассортиментом, корзиной, админ-панелью и Telegram-уведомлениями о заказах.

## Запуск

Полный запуск (сайт + бот):

```bash
./start.sh
```

Production:

```bash
./start.sh prod
```

Остановка:

```bash
./stop.sh
```

- Сайт: http://localhost:3000
- Админка: http://localhost:3000/staff-4f6867e4 (пароль меняется в самой админке, вкладка «Пароль»)

## Безопасность

- Админ-сессия подписана HMAC (`SESSION_SECRET`), куку нельзя подделать
- Cookie: `httpOnly` + `SameSite=Strict`
- SQL только через prepared statements (защита от SQL-инъекций)
- Цены заказа берутся из БД, не из браузера клиента
- Rate limit на логин и оформление заказов
- Валидация всех входных данных и заголовки CSP / X-Frame-Options
- Адрес админки — непубличный (`/staff-4f6867e4`, не угадывается и не связан со словом «admin»), закрыт от индексации (`robots.txt`) и убран из меню сайта
- `.env` в `.gitignore` — токены не коммитьте

Рекомендуется сменить `ADMIN_PASSWORD` на свой сложный пароль в `.env`.

## Telegram

1. Найдите бота в Telegram и напишите `/start` с аккаунта **@annbereg** (владелец, см. `OWNER_USERNAME` в `.env`).
2. После этого заказы будут приходить в личные сообщения — управление через кнопки внизу чата (📋 Меню, 🧾 Заказы, 👥 Менеджеры, ❓ Помощь).
3. В админке → «Менеджеры TG» можно добавить других получателей по username. Им тоже нужно написать боту `/start`.

Команды бота: `/menu`, `/orders`, `/managers`, `/help`.

## Отчёты по заказам на почту

Каждое 1-е число месяца в 06:00 бот собирает все заказы за прошлый месяц, формирует
Excel-таблицу (`.xlsx`) и отправляет её на почту, указанную в админке → «Отчёты».
Там же можно сразу отправить тестовое письмо («Отправить тестовый отчёт сейчас») —
оно соберёт заказы с начала текущего месяца по сегодня.

Отправка идёт через SMTP, настройки — в `.env`:

```
SMTP_HOST=smtp.mail.ru
SMTP_PORT=465
SMTP_USER=почта-отправитель@inbox.ru
SMTP_PASSWORD=пароль-для-внешних-приложений
```

⚠️ Планировщик отчётов встроен в Telegram-бота — работает, только пока запущен
процесс бота (`./start.sh` / PM2 `furshet-bot`). Если бот не работает 1-го числа —
отчёт за месяц не уйдёт (запустите позже — отправка вручную кнопкой в админке).

## База данных

SQLite-файл: `data/furshet.db`

- `products` — карточки ассортимента
- `orders` — история заказов
- `managers` — кто получает уведомления

Токен и пароль хранятся в `.env` (файл в `.gitignore`).

## Деплой на VPS (production)

Проект рассчитан на сервер с постоянно работающим Node.js-процессом (не serverless/Vercel) —
из-за SQLite-файла и Telegram-бота, которому нужен один живой процесс на polling.

Текущий боевой сервер: Timeweb Cloud, IP `201.24.60.88`, домен **ukusi-nsk.ru** уже
привязан (DNS A-запись → IP). Проект на сервере лежит в `~/site-`.

### 1. Подготовка сервера (один раз)

```bash
cd ~/site-
sudo bash deploy/server-setup.sh
```

Ставит Node.js 20, PM2, nginx, certbot, sqlite3 и всё нужное для сборки нативных
модулей (`build-essential`). Готовый список того, что устанавливается — прямо в
файле [`deploy/server-setup.sh`](deploy/server-setup.sh).

### 2. Код и зависимости

```bash
cd ~/site-
git pull origin main
npm ci
```

### 3. `.env`

На сервере уже должен лежать `.env` (в git не попадает). Проверьте:

- `NEXT_PUBLIC_SITE_URL=http://ukusi-nsk.ru` (после получения SSL — поменять на `https://`)
- `ADMIN_PASSWORD` — сложный пароль (или сменить прямо в админке, вкладка «Пароль»)
- `TELEGRAM_BOT_TOKEN`, `OWNER_USERNAME`, `SESSION_SECRET`, `SMTP_*` — заполнены

⚠️ Бот использует long polling — держите **только один** запущенный экземпляр бота
одновременно (или локально, или на сервере, не оба сразу), иначе Telegram вернёт
ошибку конфликта 409.

### 4. nginx + бесплатный SSL (Let's Encrypt)

```bash
cd ~/site-
sudo bash deploy/setup-nginx-ssl.sh
```

Настраивает nginx как реверс-прокси на порт 3000 для `ukusi-nsk.ru` /
`www.ukusi-nsk.ru` и сразу получает сертификат Let's Encrypt (спросит только
почту для уведомлений). Платный SSL не нужен — подробности были обсуждены
отдельно, коротко: Let's Encrypt даёт тот же уровень доверия браузеров бесплатно.

### 5. Запуск сайта и бота

Через PM2 (рекомендуется — сам поднимет процессы после падения и после
перезагрузки сервера):

```bash
npm run build
pm2 start ecosystem.config.js
pm2 save
pm2 startup   # выведет команду — выполните её, чтобы PM2 поднимался при ребуте
```

Логи: `pm2 logs furshet-web`, `pm2 logs furshet-bot`.

Либо через `./start.sh prod` (проще, но без автоперезапуска при падении процесса —
для быстрой проверки годится, для длительной работы лучше PM2).

После этого шага `http://ukusi-nsk.ru` (или `https://` после шага 4) должен
показывать сайт.

### 6. Бэкапы БД

SQLite-файл `data/furshet.db` — единственное хранилище заказов и товаров. Настройте
периодическую копию, например через cron:

```bash
0 3 * * * cp /path/to/furshet/data/furshet.db /path/to/backups/furshet-$(date +\%F).db
```
